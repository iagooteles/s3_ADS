UNIDADE I - Fundamentos de banco de dados relacional e gerenciadores de banco de dados

Banco de dados é um conjunto de software separado da aplicação que se comunica com as aplicações através da linguagem SQL.
    Consiste em um conjunto de dados inter-relacionados, agrupados internamente de acordo com seu significado no mundo real.
    Conjunto organizado de dados
    Representa informações de um mesmo domínio
    Também pode ser chamado de base de dados
    Deve possibilitar:
    ○ Armazenamento sistemático de informações;
    ○ Recuperação das informações armazenadas;
    ○ Geração de novas informações com base nos dados armazenados, a partir de questionamentos dos usuários.

- Processamento de transação online.
- Processamento para análise de dados.

SGBD => Sistema Gerenciador de Banco de Dados.
    definir, construir, manipular, compartilhar, proteger e manter múltiplos bancos de dados de forma conveniente e eficiente
ex: MySQL, Oracle, PostgreSQL, MariaDB, SQLite


Complexidades de se gerenciar um banco de dados:
● Onde armazenar os dados
● Garantir a integridade dos dados
● Permitir acesso rápido para múltiplos usuários
● Permissões de acesso


Desvantagens de não se usar um SGBD => 
- Redundância e inconsistência nos dados;
- Dificuldade no acesso aos dados;
- Isolamento dos dados;
- Problemas de integridade;
- Problemas de atomicidade => Transações atômicas(indivisíveis) => A transação será executada totalmente ou não será executada;
- Anomalias de acesso concorrente;
- Problemas de segurança;

Características e funcionalidades de um SGBD =>
• Garantir a persistência dos dados;
• Garantir a integridade e consistência dos dados;
• Definir os esquemas de bancos de dados, utilizando uma linguagem de modelagem;
• Manter uma interface através da qual as aplicações solicitarão a execução de operações e consultas aos dados;
• Ter uma linguagem de consulta, que permita a aplicação solicitar dados tanto armazenados como dados calculados ou inferidos a partir dos dados armazenados;
• Garantir a segurança de acesso aos dados;
• Ter mecanismo de backup e recuperação dos bancos de dados;
• Capacidade de replicação e distribuição dos dados;
• Controlar acessos concorrentes e transações;
• Permitir configurações de otimização de acesso aos bancos de dados, e se possível automatizar essa otimização

DBA => Data Base Administrator; 
    O DBA cria o esquema de banco de dados original executando um conjunto de instruções de definição de dados
    ● Principais atividades de um DBA:
        ○ Manutenção:
        ■ Realizar backups
        ■ Garantir que haja suficiente espaço livre em disco para operações normais e aumentar o espaço em disco, conforme necessário
        ■ Monitorar tarefas sendo executadas no banco de dados e assegurar que o desempenho não seja comprometido por tarefas muito onerosas submetidas por alguns usuários

Modelo de banco de dados =>
    ○ Modelo relacional => Este modelo usa uma coleção de tabelas para representar os dados e as relações entre eles. Cada tabela possui diversas colunas, cada qual com um único nome; Utiliza SQL; EX: Oracle, Microsoft SQL Server, PostgreSQL, MySQL;
        Organiza os dados em tabelas, formados por linhas e colunas.
    ○ Modelo de entidade/relacionamento
    ○ Modelo de dados semiestruturado => EX: Json;
    ○ Modelo de dados baseado em objeto

● Um Modelo de Dados consiste em uma coleção de diretivas, as quais permitem descrever as seguintes
características de um banco de dados:
    ○ Tipo e formato dos dados
    ○ Relacionamento entre os dados
    ○ Semântica dos dados
    ○ Restrições dos dados
● Possibilita ao administrador de dados representar o esquema de um banco, de acordo com as regras
estabelecidas pelo modelo adotado

● Com o objetivo de ocultar a complexidade de manipulação do banco de dados os projetistas de banco de
dados simplificam as interações do usuário com o sistema através de 3 níveis de abstração de dados
    ○ Esses níveis separam o usuário do banco de dados físico (arquivos armazenados em disco)
    ○ O usuário sempre trabalha com uma visão abstrata (cópia) dos dados
    ○ Detalhes de como os dados são armazenados e mantidos são ocultados dos usuários e programas
